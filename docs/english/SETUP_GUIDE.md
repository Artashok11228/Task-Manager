# Setup Guide — Task Manager

> Setup instructions based on `composer.json`, `.env.example`, and tests performed on 2026-07-12

---

## Requirements

| Software | Recommended version |
|----------|---------------------|
| PHP | 8.2 or later, with `mbstring`, `xml`, `pdo`, `sqlite` or `mysql`, `tokenizer`, `ctype`, `json`, and `bcmath` |
| Composer | 2.x |
| Node.js | 18 or later for Vite |
| npm | 9 or later |
| Database | SQLite by default, or MySQL/MariaDB |

---

## Environment Setup

### 1. Clone the Repository and Install Dependencies

```bash
composer install
npm install
```

### 2. Create the Environment File

```bash
cp .env.example .env
php artisan key:generate
```

Alternatively, use the Composer setup script:

```bash
composer run setup
```

### 3. Important Environment Variables — `.env.example`

```env
APP_NAME=Laravel
APP_ENV=local
APP_KEY=           # Generated by php artisan key:generate
APP_DEBUG=true
APP_URL=http://localhost

# Database — SQLite by default
DB_CONNECTION=sqlite

# MySQL example:
# DB_CONNECTION=mysql
# DB_HOST=127.0.0.1
# DB_PORT=3306
# DB_DATABASE=laravel
# DB_USERNAME=root
# DB_PASSWORD=

SESSION_DRIVER=database
QUEUE_CONNECTION=database
CACHE_STORE=database
MAIL_MAILER=log
FILESYSTEM_DISK=local
```

> **Note:** When using SQLite, the database file must exist at `database/database.sqlite`. Laravel's `post-create-project-cmd` script creates it automatically for a fresh project setup.

### 4. Run Migrations and Seeders

```bash
php artisan migrate
php artisan db:seed
```

Or recreate the database and seed it in one command:

```bash
php artisan migrate:fresh --seed
```

**Seeded data from `DatabaseSeeder`:**
- Four statuses: `to do`, `in Progress`, `completed`, and `hold`
- One user: `arta@gmail.com` with the password `1234`; the `User` model's `hashed` cast hashes the password

> The seeded password is suitable only for local development or demonstration. Do not use it in production.

### 5. Build Frontend Assets

```bash
# Development
npm run dev

# Production build
npm run build
```

### 6. Run the Application

```bash
# Laravel development server only
php artisan serve

# Application server, queue worker, logs, and Vite
composer run dev
```

The application is available at `http://127.0.0.1:8000` when using `php artisan serve` with its default settings.

## Queue, Cache, Session, and Storage

| Service | Default driver | Required by the current MVP? |
|---------|----------------|------------------------------|
| Queue | `database` | Optional; no application Jobs are currently defined |
| Cache | `database` | Used by Spatie Permission |
| Session | `database` | Yes, for Breeze web authentication |
| Mail | `log` | Messages are logged; no real email is sent |
| Storage | `local` | Not currently used by a feature |

To start the queue worker when future Jobs are introduced:

```bash
php artisan queue:work
```

## Accessing the API with Sanctum

### 1. Log In and Obtain a Token

```bash
curl -X POST http://127.0.0.1:8000/api/login \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"email":"arta@gmail.com","password":"1234"}'
```

### 2. Use the Token

```bash
curl http://127.0.0.1:8000/api/tasks \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: application/json"
```

## Common Setup Problems

| Problem | Resolution |
|---------|------------|
| `SQLSTATE[HY000] database file does not exist` | Run `touch database/database.sqlite` |
| Vite assets cannot be found | Run `npm run build` or keep `npm run dev` running |
| HTTP 419 on web forms | Verify the session configuration and run `php artisan migrate` so the `sessions` table exists |
| Permission cache error | Run `php artisan config:clear`, then `php artisan migrate` |
| Dashboard redirects to email verification | The user may not have an `email_verified_at` value; use `/verify-email` or create a verified user through the factory |
